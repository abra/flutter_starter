# Localization guide

How locale selection works, and how to add translated strings to the app.

---

## How locale currently works

```
AppSettings.locale          ← user's selected Locale stored in SharedPreferences
       ↓
AppSettingsScope            ← streams AppSettings through the widget tree
       ↓
MaterialApp.locale          ← MaterialApp uses it to resolve the active locale
       ↓
Flutter widget tree         ← standard Flutter widgets (dates, pickers, etc.) adapt automatically
```

**`AppSettings` does not need to change** when adding more languages —
it already stores a `Locale` value. The only question is which locales you
make available to the user.

---

## Step 1 — Add flutter_localizations to pubspec.yaml

```yaml
dependencies:
  flutter:
    sdk: flutter
  flutter_localizations:      # ← add this
    sdk: flutter
```

---

## Step 2 — Register supported locales in MaterialApp

`lib/app/material_context.dart`:

```dart
import 'package:flutter_localizations/flutter_localizations.dart';

MaterialApp(
  locale: settings.locale,

  supportedLocales: const [
    Locale('en'),
    Locale('ru'),
    // add more here as you create ARB files for them
  ],

  localizationsDelegates: [
    GlobalMaterialLocalizations.delegate,     // Material widget strings
    GlobalWidgetsLocalizations.delegate,      // Text direction
    GlobalCupertinoLocalizations.delegate,    // Cupertino widget strings

    // Feature delegates go here (Step 5)
  ],

  ...
)
```

`supportedLocales` tells Flutter which languages the app can display.
If the user's device language is not in this list, Flutter falls back to
the first entry (`en` in the example above).

---

## Step 3 — Enable code generation

Add `generate: true` under `flutter:` in the root `pubspec.yaml`:

```yaml
flutter:
  generate: true
```

Create `l10n.yaml` at the project root:

```yaml
arb-dir: lib/l10n
template-arb-file: intl_en.arb
output-localization-file: app_localizations.dart
```

---

## Step 4 — Create ARB files

`lib/l10n/intl_en.arb`:

```json
{
  "@@locale": "en",
  "appTitle": "My App",
  "@appTitle": { "description": "The application title" }
}
```

`lib/l10n/intl_ru.arb`:

```json
{
  "@@locale": "ru",
  "appTitle": "Моё приложение"
}
```

Run generation (or let the IDE do it automatically):

```bash
flutter gen-l10n
```

This generates `.dart_tool/flutter_gen/gen_l10n/app_localizations.dart`
with an `AppLocalizations` class and an `AppLocalizations.delegate`.

Use in widgets:

```dart
import 'package:flutter_gen/gen_l10n/app_localizations.dart';

final l10n = AppLocalizations.of(context)!;
Text(l10n.appTitle)
```

Add the delegate to `MaterialApp`:

```dart
localizationsDelegates: [
  AppLocalizations.delegate,            // ← generated delegate
  GlobalMaterialLocalizations.delegate,
  GlobalWidgetsLocalizations.delegate,
  GlobalCupertinoLocalizations.delegate,
],
```

## Step 5 — Localization in feature packages

Each feature package can manage its own translations independently.

### Directory structure

```
packages/features/notes/
  lib/
    l10n/
      intl_en.arb
      intl_ru.arb
  l10n.yaml                 ← package-level l10n config
```

`packages/features/notes/l10n.yaml`:

```yaml
arb-dir: lib/l10n
template-arb-file: intl_en.arb
output-localization-file: notes_localizations.dart
output-class: NotesLocalizations
```

Run generation from the package directory:

```bash
cd packages/features/notes && flutter gen-l10n
```

### Register the feature delegate

In `lib/app/material_context.dart` add the feature's delegate to the list:

```dart
import 'package:notes/src/l10n/notes_localizations.dart';

localizationsDelegates: [
  NotesLocalizations.delegate,          // ← feature delegate
  AppLocalizations.delegate,
  GlobalMaterialLocalizations.delegate,
  GlobalWidgetsLocalizations.delegate,
  GlobalCupertinoLocalizations.delegate,
],
```

Use in the feature's widgets:

```dart
final l10n = NotesLocalizations.of(context)!;
Text(l10n.emptyStateMessage)
```

## Step 6 — Let the user pick a locale

The locale picker lives in your settings screen, not in `app_settings` package.
Call `update` on the service to persist the selection:

```dart
// somewhere in your settings screen
AppSettingsScope.of(context).settingsService.update(
  (s) => s.copyWith(locale: const Locale('ru')),
);
```

The change is persisted immediately and `AppSettingsBuilder` in `MaterialContext`
rebuilds `MaterialApp` with the new locale.

---

## Locale fallback rules

Flutter resolves the active locale with this priority:

1. `MaterialApp.locale` — the value from `AppSettings` (user's explicit choice)
2. Device locale — used when `locale` is null
3. First entry in `supportedLocales` — last resort fallback

If the user selects a locale that is not in `supportedLocales`, Flutter logs a
warning and uses the closest match or the fallback. Always keep `supportedLocales`
in sync with the ARB files you actually ship.
